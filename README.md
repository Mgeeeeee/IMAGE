# 项目主文档

一旦我所属的文件夹有所变化，请更新我。
任何功能、架构、写法更新，必须在工作结束后同步更新相关目录的子文档。

架构：`index.html` 为入口，`assets/` 承载样式与脚本。  
文档：`docs/` 管理使用、交接、规范与记录。  
规范与提示词：`meta/` 约束协作，`Prompts/` 仅供参考。  

文件清单（名字 / 地位 / 功能）：
- `.gitignore` — 地位：协作保护；功能：忽略系统与编辑器噪音文件。
- `index.html` — 地位：运行入口；功能：结构承载与资源加载。
- `assets/` — 地位：资源中心；功能：样式与脚本入口。
- `docs/` — 地位：文档中心；功能：使用、交接、规范与参考记录。
- `meta/` — 地位：规范中心；功能：身份与贡献约束。
- `Prompts/` — 地位：提示词库；功能：人工参考的预设提示词。
- `README.md` — 地位：根目录主文档；功能：规则与索引入口。

更多细节见 `docs/README.md` 与 `meta/AGENTS.md`。
