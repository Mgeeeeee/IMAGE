# assets 目录

一旦我所属的文件夹有所变化，请更新我。

`assets/` 存放样式与脚本资源。  
只服务 `index.html` 的加载与渲染。  
保持静态托管的可访问性。  

触发协议：新增/移动/删除/重命名文件 → 更新本文件 → 同步上一级目录索引。
[PROTOCOL]: 变更时更新此头部，然后检查 AGENTS.md

文件清单（名字 / 地位 / 功能）：
- `1.png` — 地位：视觉参考；功能：本地示意图片。
- `styles.css` — 地位：样式入口；功能：全站视觉与布局。
- `app.js` — 地位：脚本入口；功能：交互与生成逻辑。
- `prompt-utils.js` — 地位：文本清洗；功能：提示词规范化。
- `image-api.js` — 地位：请求封装；功能：生图路由与响应解析。
- `storage.js` — 地位：缓存管理；功能：配置持久化读写。
- `ui-state.js` — 地位：状态管理；功能：面板与加载等 UI 控制。
- `generation.js` — 地位：流程编排；功能：生成链路与状态协调。
- `favicon.svg` — 地位：站点图标；功能：书签与标签页展示。
- `apple-touch-icon.png` — 地位：触控图标；功能：移动端书签与桌面图标。
- `svgviewer-png-output.png` — 地位：视觉参考；功能：本地图标导出物。
- `FOLDER.md` — 地位：目录索引；功能：列出 assets 内容与职责。
- `AGENTS.md` — 地位：模块地图；功能：assets 架构文档。
